### `scripts/prepare_solr_docs.py` — rewritten as a four-layer pipeline

The monolithic script was replaced with a clearly structured pipeline.

#### Layer 1 — Loaders (per-source CSV parsing)

**`load_31_csv(csv_path, source_dataset)`** — handles the three 3.1-format files:
- Reads columns `Post/Comment text`, `Number of upvotes`, `Posted time`, `Subreddit the post/comment is from`
- Sets `type = "unknown"` for all rows — the 3.1 CSVs have no structural type signal, and length-based inference has been removed
- `title` is always empty for 3.1 docs; `body` and `search_text` both hold the full cleaned text

**`load_sentiment_csv(csv_path)`** — handles `reddit_ai_sentiment_shortened.csv`:
- Reads columns `id`, `text`, `subreddit`, `type`, `score`, `created_utc`
- Strips BOM via `encoding="utf-8-sig"` and detects/skips a stray non-header first line
- Preserves explicit Reddit IDs as the stable `id` (e.g. `144lfc1`) rather than hashing
- Converts Unix epoch `created_utc` to UTC ISO-8601 via `to_solr_date()`
- Honours the real `type` field (`post` / `comment`) from the file
- For posts: derives `title` as first 150 characters; `search_text = title + " " + body`

#### Layer 2 — Normalizer

Both loaders emit the same `SolrDoc` dataclass (19 fields). The normalizer:
- Calls `to_solr_date()` on all timestamps — handles both Unix epoch integers and ISO-8601 strings, always outputs `YYYY-MM-DDTHH:MM:SSZ`
- Strips BOM characters from text
- Removes `[deleted]` / `[removed]` markers and collapses whitespace
- Computes `upvote_log = log(max(score, 0) + 1)`
- Classifies `time_bucket` by comparing `created_date` to the current date at index time

#### Layer 3 — Enrichment

Three functions run on every document's `body` text:

**`extract_models(text)`** — regex matching against 9 canonical model names and their aliases:

| Canonical | Aliases matched |
|---|---|
| `chatgpt` | chatgpt, chat gpt, gpt-4, gpt4, gpt-3, gpt3, gpt 4, gpt 3 |
| `claude` | claude |
| `gemini` | gemini, bard |
| `llama` | llama, llama2, llama 2, llama3, llama 3 |
| `copilot` | copilot, github copilot |
| `mistral` | mistral |
| `grok` | grok |
| `deepseek` | deepseek, deep seek |
| `perplexity` | perplexity |
| `palm` | palm, palm2, palm 2 |

**`extract_vendors(text)`** — regex matching against 7 canonical vendor names:

| Canonical | Aliases matched |
|---|---|
| `openai` | openai, open ai |
| `anthropic` | anthropic |
| `google` | google deepmind, google ai, deepmind |
| `meta` | meta ai, meta llama |
| `microsoft` | microsoft |
| `mistral_ai` | mistral ai |
| `xai` | x.ai, xai |

**`compute_sentiment(text)`** — lexical positive/negative word counting:
- Counts matches against positive and negative word lists
- Returns `neutral` when no opinion words are found
- Returns `mixed` when both polarities are present and the ratio is within 40% of balanced
- Returns `positive` or `negative` when one polarity dominates
- `sentiment_score` is the normalised ratio clamped to `[-1.0, 1.0]`

**`compute_opinionatedness(text)`** — counts first-person and evaluative markers
(e.g. "I think", "IMO", "honestly", "terrible", "amazing") relative to document length,
returning a score in `[0.0, 1.0]`.

#### Layer 4 — Serializer

- Writes one JSON object per line to the output JSONL file
- Deduplication key changed to `(body.lower()[:200], subreddit, created_date)` — uses the first 200 characters of body rather than `full_text` to avoid false misses from the title removal
- Empty `model_mentions` and `vendor_mentions` lists are omitted from the output dict to avoid Solr indexing empty multi-value fields

**New CLI flag:** `--sentiment-file <path>` — pass the path to `reddit_ai_sentiment_shortened.csv` explicitly. If omitted, the script auto-detects it by checking three common relative paths.

| Parameter | Before | After |
|---|---|---|

**Retrieved fields** expanded to include:
`source_dataset, sentiment_label, sentiment_score, model_mentions, vendor_mentions, opinionatedness_score`

**Highlighting fields** changed from `title, body, full_text` to `search_text, body, title`
(matching the renamed combined field).
