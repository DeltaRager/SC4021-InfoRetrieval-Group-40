## Changes

### `nlp_utils.py` — new file

A self-contained NLP utility module providing two public functions:

**`process_query(query)`** — used at query time by `app.py`:
- Detects and expands prefix/wildcard tokens (e.g. `artif` → `artificial`, `int*enc` → `int*enc` passed to Solr as-is)
- Spell-corrects individual tokens using `pyspellchecker`; preserves known domain terms (model names, etc.)
- Lemmatizes the corrected query with spaCy (`en_core_web_sm`)
- Extracts multi-word concept phrases via YAKE keyphrase extraction + spaCy noun chunks + NER
- Returns a dict with keys: `original`, `expanded`, `spell_corrected`, `final_query`, `lemmatized`,
  `cleaned`, `concepts`, `has_typo`, `has_expansion`, `corrections`, `expansions`, `fuzzy`

**`process_for_indexing(text)`** — used at index time by `prepare_solr_docs.py`:
- Runs lemmatization and concept extraction on a document's text
- Returns `{"lemmatized_text": str, "concepts": str}` for storage in Solr

### `schema_add_fields.json` — two NLP fields added

| Field | Type | Purpose |
|---|---|---|
| `lemmatized_text` | text_en | spaCy-lemmatized form of `search_text`; improves morphological recall |
| `concepts` | text_en | YAKE + noun chunk + NER keyphrases extracted at index time |

Copy-field directives extended to copy `lemmatized_text` and `concepts` into `_text_` alongside the
existing `title`, `body`, and `search_text` directives.

### `scripts/prepare_solr_docs.py` — NLP enrichment added

`SolrDoc` dataclass gains two new fields: `lemmatized_text` and `concepts`.

A new `enrich_nlp(text)` helper wraps `process_for_indexing` with a graceful fallback: if
`nlp_utils` is not importable (missing spaCy model or YAKE), both fields are set to empty strings
and the pipeline continues without error.

Both loaders (`load_31_csv` and `load_sentiment_csv`) now call `enrich_nlp(search_text)` on each
document. `to_dict()` omits empty `lemmatized_text` and `concepts` keys so Solr does not index
empty fields.

A warning is printed at startup if `nlp_utils` is unavailable:
```
[WARN] nlp_utils not available; lemmatized_text and concepts will be empty.
```

### `app.py` — NLP query pipeline wired in

**New import:** `from nlp_utils import process_query`

**New query parameter:** `nlp` — checkbox toggle; defaults to enabled (`use_nlp = True`).
The form uses a hidden `value="0"` + checkbox `value="1"` pattern so the server can detect the
unchecked state even when the form omits the checkbox value.

**Query flow when NLP is enabled:**
1. `process_query(q)` runs prefix expansion, spell correction, and lemmatization; returns `nlp_info`.
2. `solr_q` is replaced with `nlp_result["final_query"]` (auto-corrected, never a "Did you mean" prompt).
3. `qf` is extended: `title^4 search_text^3 body^1.5 lemmatized_text^2.5 concepts^2`
4. `pf` is extended: `title^8 search_text^4 lemmatized_text^3`
5. Boost queries (`bq`) are added for lemmatized form (^2), concept phrases (^1.5), and fuzzy
   original (^1.2) — these only affect ranking, never exclude documents.

**Highlighting fields** extended to `search_text, body, title, lemmatized_text, concepts`.

**Template variables added:** `use_nlp`, `nlp_info` passed to `render_template`.

### `templates/index.html` — NLP UI elements added

**NLP toggle** — a checkbox below the filter row:
```
☑ Enable NLP enhancement (lemmatization, concept extraction, spell correction, prefix/wildcard)
```

**Auto-correction bar** — shown when `nlp_info.has_typo` or `nlp_info.has_expansion` is set;
displays a struck-through original token and a green corrected form for each substitution.

**NLP analysis panel** — shown when NLP is enabled and a query was processed; displays:
- Original query
- After prefix/wildcard expansion (if any)
- After spell correction (if any)
- Final query sent to Solr
- Lemmatized form
- After stopword removal
- Extracted concept tags (up to 8, rendered as blue pill badges)

Both panels are hidden when NLP is disabled or when no query has been submitted.
