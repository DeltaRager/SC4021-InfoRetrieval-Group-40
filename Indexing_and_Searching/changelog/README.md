# Indexing and Searching Changelog

Canonical changelog entries for `Indexing_and_Searching/` live in this directory.

## Layout

- Use `YYYY-MM-DD/HHMMSS.md` for every entry.
- One file represents one work session or commit batch.
- Multiple related commits may appear in the same file.
- Timestamps use `Asia/Singapore` time.

## Entry Template

```md
# <Session title>

- Timestamp: YYYY-MM-DD HH:MM:SS SGT
- Commits: `abc1234`, `def5678`

## Summary

Short description of the session outcome.

## Changes

### `<file or area>`

What changed and why.

## Notes

Optional extra context, caveats, or follow-up details.
```

## Historical Migration

The original monolithic `CHANGELOG.md` was split into timestamped session files on 2026-04-10.
